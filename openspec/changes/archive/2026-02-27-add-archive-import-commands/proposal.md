## Why

当前 `fs` 工具支持多环境配置管理，但缺乏配置的导出和导入功能。用户在不同环境间迁移配置或与团队成员共享主机配置时，需要手动复制多个 YAML 文件，过程繁琐且容易出错。需要提供标准化的配置打包和导入机制，提升配置管理的便捷性和可移植性。

## What Changes

- 新增 `archive` 命令：将当前环境的主机配置打包成单个文件，便于分发和备份
- 新增 `import` 命令：将打包的配置文件导入到当前环境
- 支持配置冲突处理机制
- 提供打包文件的验证和安全性检查

## Capabilities

### New Capabilities
- `config-archive`: 实现主机配置的打包功能，支持多种输出格式（tar.gz）
- `config-import`: 实现打包配置文件的导入功能，包含冲突检测和处理

### Modified Capabilities
- 无

## Impact

- 新增两个命令行子命令
- 需要处理文件压缩/解压缩操作
- 涉及 YAML 配置文件的序列化和反序列化
- 需要考虑配置导入时的冲突处理策略
- 可能需要更新文档和帮助信息
